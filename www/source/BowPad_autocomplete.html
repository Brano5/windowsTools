auto-completion and code snippets

BowPad can scan files for function names and other words to be used in auto completion. While the automatic scan does a well enough job, sometimes you might want to provide your own word list for auto completion.

To do that, create a file named autocomplete.ini in %APPDATA%\BowPad if you have BowPad installed, or in in the bin directory for portable versions of BowPad.

BowPad can also insert code snippets. For example if you type do in a C/C++ file, you'll get a suggestion popup to insert a do-while code loop. Custom code snippets can be added to the same autocomplete.ini file.

The format of that file is: 

[language]
code=word1 word2 word3
snippet_do=do\n{\n} while(^0true^);
snippet_for=for (^1size_t^ ^2i^ = 0; ^2i^ < ^3count^; ++^2i^)\n{\n\t^0^\b}
snippet_while=while (^0true^)\n{\n\t^0^\b}
snippet_class=class ^0MyClass^\n{\npublic:\n\t^0MyClass^();\n~^0MyClass^();\n\b\n\bprotected:\n\nprivate:\n\n};\n
snippet_enum=enum ^0MyEnum^\n{\n\t^0^\b};\n
snippet_switch=switch (^0switch_on^)\n{\tdefault:\nbreak;\n}\n
snippet_try=try\n{\n\t^0^\b}\ncatch (const std::exception&)\n{\n}\n

The [language] must be replaced with the lexer for which these settings apply, so e.g. C/C++, C#, Ruby, ...

The code line contains a space separated list of words used for auto completion.

The snippet_ lines contain the code snippets. The part after the _ char is the snippet trigger, and the value is the string/snippet to insert. The ^nTEXT^ marks an insert point where e.g. a variable can be modified. The insert point 0 is special: it marks the position where the cursor is placed after the snippet has been inserted fully. So use 1...9 for the real insert points. The user can then tab through those insert points until the end. The n marks the number of the insertion point. See the snippet_for example above.

You can use \n to insert newlines, \t to insert a tab, and \b to insert a backspace. Since the ^ char is used to mark insertion points, you can use \^ to insert a real ^ char.
